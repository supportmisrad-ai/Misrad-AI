# 💰 Finance — אינטגרציית כספים וחשבוניות

> 🎁 **מודול זה ניתן במתנה לכל לקוח — בכל חבילה!**  
> **משתמשים:** 1 (סולו) או 5 (בחבילה)  
> **מהות:** שכבת אינטגרציה חכמה מול חברות חשבוניות חיצוניות דרך API  
> **המודול לא מחליף רואה חשבון** — הוא מחבר אותך לשירותי החשבוניות שלך ומאפשר לנהל הכל ממקום אחד.
> 
> **למה במתנה?** כי כל עסק צריך לנהל כספים. אנחנו לא רוצים שתשלם על הבסיס.

---

## ⚡ איך זה עובד?

מודול Finance הוא **לא מערכת הנהלת חשבונות**. הוא שכבת ממשק (UI/UX) שמתחברת דרך API לחברות חשבוניות חיצוניות שאתה כבר משתמש בהן:

```
┌─────────────────────────────────┐
│  MISRAD AI — מודול Finance      │
│  (ממשק אחיד, חכם ומרכזי)       │
└────────────┬────────────────────┘
             │ API
   ┌─────────┼─────────┬──────────┬──────────┐
   ▼         ▼         ▼          ▼          ▼
Morning   iCount    חשבוניות   סמיט     EZcount
(חשבונית           אונליין
 ירוקה)
```

**אתה בוחר את חברת החשבוניות שלך** → MISRAD AI מתחבר אליה → ומאפשר לך לעבוד מתוך המערכת בלי לפתוח חלון נפרד.

## 🖥️ מסכי UI/UX

### 1. לוח בקרה כספי

| רכיב | תיאור |
|-------|--------|
| **סיכום חודשי** | סה"כ הכנסות, הוצאות, יתרה — נשלפים מהחברה המחוברת |
| **חובות פתוחים** | רשימת חשבוניות שלא שולמו + ימי איחור |
| **גרף מגמות** | הכנסות/הוצאות לאורך זמן |
| **התראות** | חשבוניות באיחור, תזכורות שנכשלו |

### 2. חשבוניות (דרך API)

| פעולה | תיאור |
|--------|--------|
| **יצירה מהירה** | לקוח + פריטים + סכום → נשלח לחברת החשבוניות דרך API |
| **סטטוסים** | טיוטה → נשלחה → שולמה / באיחור / בוטלה |
| **סנכרון** | כל שינוי במערכת ← → מתעדכן בחברת החשבוניות ולהפך |
| **PDF** | הורדת PDF ישירות מחברת החשבוניות |
| **שליחה** | שליחה ללקוח במייל או בוואטסאפ |
| **מספור** | מספור אוטומטי לפי חברת החשבוניות |

**סטטוסים:**
```
טיוטה → נשלחה → שולמה ✅
                → באיחור ⚠️ (עבר תאריך פירעון)
                → בוטלה ❌
```

### 3. תזכורות תשלום בוואטסאפ ⭐

| פיצ'ר | תיאור |
|--------|--------|
| **תזכורת אוטומטית** | X ימים לפני/אחרי תאריך פירעון |
| **התאמה אישית** | תבנית הודעה בעברית מותאמת |
| **לינק לתשלום** | קישור ישיר לתשלום (PayMe/Meshulam/Hyp) |
| **מעקב סטטוס** | נשלח / נקרא / נכשל |
| **היסטוריה מלאה** | כל התזכורות שנשלחו |

**דוגמה:**
```
"שלום [שם], תזכורת: חשבונית #1234 בסך ₪5,000 
 לתשלום עד 15/02/2026. 
 לתשלום מיידי: [קישור לתשלום]"
```

### 4. הוצאות

| פיצ'ר | תיאור |
|--------|--------|
| **רישום מהיר** | סכום + קטגוריה + תיאור |
| **קטגוריות** | משרד, נסיעות, שיווק, ציוד... |
| **צירוף קבלה** | העלאת תמונה/PDF |
| **הוצאה חוזרת** | הוצאה קבועה (שכירות, ביטוח...) |
| **סנכרון** | סנכרון הוצאות עם חברת ההנהלת חשבונות |

### 5. דוחות

| דוח | תיאור |
|------|--------|
| **הכנסות / הוצאות** | סיכום חודשי + גרף (נשלף מהחברה) |
| **לפי לקוח** | כמה כל לקוח שילם |
| **חובות פתוחים** | מי חייב כמה + ימי איחור |
| **מע"מ** | סיכום מע"מ לתקופה (מחברת החשבוניות) |
| **ייצוא** | Excel / CSV / PDF |

### 6. הגדרות אינטגרציה

| הגדרה | תיאור |
|--------|--------|
| **בחירת ספק** | בחר חברת חשבוניות (Morning/iCount/סמיט/...) |
| **מפתח API** | הזנת API Key מחברת החשבוניות |
| **בדיקת חיבור** | בדיקה שהחיבור פעיל ותקין |
| **סנכרון ראשוני** | ייבוא חשבוניות קיימות מהחברה |
| **הגדרות ברירת מחדל** | מטבע, מע"מ, תנאי תשלום |

## 🔗 חיבורים בין-מודולריים

מודול Finance משולב עמוק עם שאר המודולים במערכת:

### Finance ← Nexus (ניהול צוות)
**מקור נתונים:** `nexus_time_entries` (דיווחי זמן)

| נתון | שימוש ב-Finance |
|------|------------------|
| **שעות עבודה** | חישוב עלות עבודה (Labor Cost) לפי שעתי/חודשי |
| **דיווחי זמן** | סיכום הוצאות עבודה לפי עובד/מחלקה/פרויקט |
| **שכר עובדים** | תחשיב רווחיות: הכנסות מול עלות עבודה |

**דוגמה:** מסך הוצאות (`/finance/expenses`) מציג עלות עבודה מצטברת על בסיס `nexusTimeEntry.durationMinutes` כפול תעריף שעתי/חודשי.

---

### Finance ← System (מכירות)
**מקור נתונים:** `system_invoices`, `system_leads`

| נתון | שימוש ב-Finance |
|------|------------------|
| **חשבוניות System** | תחזית הכנסות חודשית (pending invoices) |
| **Pipeline משוקלל** | חיזוי הכנסות עתידיות לפי שלב מכירה |
| **Lead → Invoice** | כשליד הופך ל-Won, נוצר `SystemInvoice` אוטומטית |

**דוגמה:** דוח תחזית הכנסות (`/api/workspaces/[orgSlug]/me-insights`) מחשב: `weightedPipeline + systemInvoicesOpen + misradInvoicesOpen`.

---

### Finance ← Social (לקוחות שיווק)
**מקור נתונים:** `social_clients`, `social_payment_orders`

| נתון | שימוש ב-Finance |
|------|------------------|
| **הזמנות תשלום** | Portal לקוח יוצר `social_payment_orders` |
| **מנוי חודשי** | `monthly_fee` + `next_payment_date` לתחזית הכנסות חוזרות |
| **סטטוס תשלום** | `payment_status` מסונכרן עם Finance |

**דוגמה:** לקוח Social שמשלם דרך Portal → נוצר payment order → Finance מעדכן סטטוס → אפשר ליצור חשבונית Morning.

---

### Finance ← Client (פורטל לקוחות)
**מקור נתונים:** `client_clients` (לקוחות B2B)

| נתון | שימוש ב-Finance |
|------|------------------|
| **פרטי לקוח** | מילוי אוטומטי בחשבונית (שם, אימייל, טלפון) |
| **חשבוניות ללקוח** | `ClientDetailModal` מציג היסטוריית חיובים + כפתור "הפק חשבונית" |
| **Portal Workflow** | לקוח יכול לאשר תוכן → Finance יוצר חשבונית אוטומטית |

**דוגמה:** `ClientDetailModal.tsx` (טאב חשבוניות) פותח `CreateInvoiceModal` עם פרטי הלקוח מראש.

---

### Finance ← Operations (תפעול ושטח)
**מקור נתונים:** `operations_work_orders`, `operations_projects`

| נתון | שימוש ב-Finance |
|------|------------------|
| **קריאות שירות** | Work Order שהושלם → אפשרות ליצור חשבונית אוטומטית |
| **פרויקטים** | מעקב רווחיות: הכנסות מהפרויקט מול עלות עבודה ומלאי |
| **חלקים ומלאי** | `operations_stock_movements` מחושב כהוצאה ישירה |

**דוגמה:** טכנאי סוגר Work Order → אפשר ליצור חשבונית ללקוח ישירות עם פירוט החלקים והעבודה.

---

### Finance → כל המודולים (יצירת חשבוניות)

**Finance מספק ליכולת יצירת חשבוניות לכל המודולים:**

```typescript
// דוגמה: יצירת חשבונית מכל מודול
import { createInvoice } from '@/lib/integrations/green-invoice';

// Nexus: חשבונית עבור פרויקט
const projectInvoice = await createInvoice(userId, {
  clientName: project.clientName,
  items: [{ description: 'פרויקט X', quantity: 1, price: 5000 }],
}, organizationId);

// System: חשבונית אחרי סגירת עסקה
const leadInvoice = await createInvoice(userId, {
  clientName: lead.companyName,
  items: [{ description: lead.serviceDescription, quantity: 1, price: lead.dealValue }],
}, organizationId);

// Operations: חשבונית אחרי Work Order
const woInvoice = await createInvoice(userId, {
  clientName: workOrder.clientName,
  items: workOrder.partsUsed.map(p => ({ description: p.name, quantity: p.qty, price: p.price })),
}, organizationId);
```

```sql

misrad_invoices:
  id, organization_id, client_id

  id, organization_id, lead_id

  → נוצר אוטומטית כש-Lead הופך ל-Won

finance_whatsapp_reminder_events:
  id, organization_id, invoice_id, client_id
  sent_at, delivery_status, message_template, phone
  → מחובר ל-misrad_invoices

-- אינטגרציות (API Keys מוצפנים)
scale_integrations:
  id, user_id, tenant_id, service_type
  access_token (encrypted), is_active, last_synced_at
  metadata JSONB (usage tracking)
  → service_type = 'green_invoice' (Morning)

-- הוצאות (מחושב מדיווחי זמן)
nexus_time_entries:
  id, organizationId, userId, date, durationMinutes
  → Finance מחשב: totalCost = Σ(minutes × hourlyRate)

-- הוצאות ישירות (מלקוחות)
misrad_clients.directExpenses:
  → נצבר בשדה directExpenses בלקוח
  → Finance מציג: totalExpenses = laborCost + directExpenses

-- הזמנות תשלום (Portal לקוחות)
social_payment_orders:
  id, client_id, amount, description
  status (pending/paid), installments_allowed
  → לאחר תשלום → אפשר ליצור חשבונית Morning
```

## תהליך עבודה

### תהליך: חשבונית + תזכורת + תשלום
```
1. בחר לקוח + פריטים → צור חשבונית ב-MISRAD AI
2. המערכת שולחת דרך API לחברת החשבוניות (Morning/iCount/...)
3. חברת החשבוניות יוצרת חשבונית מס רשמית
4. חשבונית נשלחת ללקוח במייל + לינק לתשלום
5. 3 ימים לפני תאריך פירעון → תזכורת וואטסאפ אוטומטית
6. יום התשלום → תזכורת נוספת אם לא שולם
7. לקוח שילם → סנכרון סטטוס אוטומטי מהחברה
8. לא שילם → סימון "באיחור" + התראה למנהל
```
