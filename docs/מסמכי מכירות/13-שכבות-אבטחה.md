**מה זה מגן:**

- 20 ניסיונות → 24 שעות

## שכבה 2: Tenant Guard

**מה זה:**
Middleware שמסנן אוטומטית לפי `organization_id`.

**קוד:**
```typescript
async function getTasks() {
  const orgId = await getOrganizationId();
  
  return prisma.task.findMany({
    where: { organizationId: orgId }
  });
}
```

**מה קורה אם מנסים לעקוף:**
```typescript
// ניסיון גישה לארגון אחר
prisma.task.findMany({
  where: { organizationId: "other-org-id" }
});

// תוצאה: [] (ריק)
// למה? Guard מוסיף תנאי נוסף שתמיד false
```

## שכבה 3: Row Level Security (RLS)

**מה זה:**
הגנה ב-Database עצמו.

**קוד SQL:**
```sql
CREATE POLICY "org_isolation"
ON nexus_tasks
USING (organization_id = current_setting('app.organization_id')::uuid);

ALTER TABLE nexus_tasks ENABLE ROW LEVEL SECURITY;
```

**משמעות:**
גם אם אני אכתוב SQL ישיר, PostgreSQL יסנן!

## שכבה 4: API Guards

**מה זה:**
בדיקות בכל Server Action.

**קוד:**
```typescript
export async function createTask(data) {
  // 1. אימות
  const session = await auth();
  if (!session) throw new Error('Unauthorized');
  
  // 2. ארגון
  const orgId = await getOrganizationId();
  if (!orgId) throw new Error('No org');
  
  // 3. הרשאות
  const canCreate = await checkPermission('create_task');
  if (!canCreate) throw new Error('Forbidden');
  
  // 4. Rate Limit
  const allowed = await checkRateLimit('create_task');
  if (!allowed) throw new Error('Rate limit');
  
  // 5. יצירה
  return prisma.task.create({
    data: { ...data, organizationId: orgId }
  });
}
```

## שכבה 5: AI Security

**סינון נתונים רגישים:**
```typescript
const SENSITIVE_FIELDS = [
  'email', 'phone', 'password',
  'hourlyRate', 'monthlySalary',
  'creditCard', 'ssn', 'idNumber'
];

function sanitizeForAI(data) {
  return filterFields(data, SENSITIVE_FIELDS);
}
```

**דוגמה:**
```typescript
// לפני שליחה ל-AI
const user = { 
  name: "John", 
  email: "john@example.com",  // 🔒 רגיש
  salary: 15000  // 🔒 רגיש
};

const sanitized = sanitizeForAI(user);
// { name: "John" }
```

## Incident Response

**אם יש חשד לפריצה:**

1. **זיהוי** (תוך דקות)
   - התראה אוטומטית
   - ניתוח logs

2. **בידוד** (תוך שעה)
   - חסימת חשבון חשוד
   - סגירת גישות

3. **תיקון** (תוך 24 שעות)
   - Patch אבטחה
   - איפוס סיסמאות

4. **דיווח** (תוך 72 שעות)
   - הודעה ללקוחות מושפעים
   - דיווח לרשויות (אם נדרש)
