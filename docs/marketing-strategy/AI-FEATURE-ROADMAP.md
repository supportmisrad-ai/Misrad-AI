# 🤖 AI Feature Roadmap — פיצ'רים חסרים שצריך לבנות

> **מטרה:** לבנות את כל יכולות ה-AI שהוזכרו בחומרים השיווקיים — כדי שהשיווק יהיה **אמיתי**.
> **עיקרון:** כל פיצ'ר חייב לעבוד באמת לפני שמפרסמים אותו.

---

## 📊 סיכום מצב נוכחי

### ✅ קיים ועובד
1. ניתוח שיחות מכירה (ציון, סנטימנט, התנגדויות, המלצות)
2. Lead Scoring (AI Score 0-100)
3. Content Machine (AI מייצר וריאציות לפוסטים)
4. Voice Commands בעברית (הקלטה → תמלול → פעולה)
5. AI Assistant (שאלות חופשיות בכל מסך)
6. Health Score מכירות (ציון בריאות)
7. AI Insights מכירות (תובנות על conversion, מקורות)
8. אבטחת AI (סינון שדות, audit log)
9. Semantic Search (חיפוש מבוסס embeddings)
10. Multi-provider (Gemini, OpenAI, Anthropic, Groq, Deepgram)

### ❌ חסר — צריך לבנות (9 פיצ'רים)

## 🔥 Feature 1: חיזוי סגירות (Deal Closure Prediction)

**מודול:** System (CRM מכירות)
**מה חסר:** Lead Scoring נותן ציון כללי, אבל אין חיזוי סגירה באחוזים עם הסבר
**מה צריך לבנות:**

### Backend
- [ ] הרחבת `recomputeSystemLeadAiScore` ב-`app/actions/system-leads.ts`
- [ ] הוספת שדות ל-SystemLead: `closureProbability` (number 0-100), `closureRationale` (text)
- [ ] Prompt חדש שמחזיר JSON:
```json
{
  "score": 85,
  "isHot": true,
  "closureProbability": 75,
  "closureRationale": "הלקוח ביקש הצעת מחיר, שיחה חיובית, שאל על לוחות זמנים",
  "recommendedAction": "שלח הצעה עם אופציה חודשית עד יום רביעי",
  "tags": ["מתעניין", "הצעת מחיר", "סגירה קרובה"]
}
```

### UI
- [ ] תצוגת אחוז סגירה בכרטיס ליד (`LeadBusinessSide.tsx`)
- [ ] Badge צבעוני: 0-30% אדום, 31-60% כתום, 61-100% ירוק
- [ ] Tooltip עם הסבר (rationale)
- [ ] כפתור "מה לעשות?" שמראה המלצת AI

### UX
- [ ] אחוז הסגירה מתעדכן אוטומטית אחרי כל אינטראקציה
- [ ] Pipeline view עם מיון לפי סיכוי סגירה
- [ ] התראה כשליד חם (>70%) לא טופל 48 שעות

**מורכבות:** בינונית (הרחבה של פיצ'ר קיים)
**זמן משוער:** 1-2 ימים

## 🔥 Feature 2: ניתוח רווחיות פרויקט (Project Profitability Analysis)

**מודול:** Finance
**מה חסר:** Finance לא מנתח רווחיות per-project/per-client עם AI
**מה צריך לבנות:**

### Backend
- [ ] Server Action חדש: `analyzeProjectProfitability` ב-`app/actions/finance-actions.ts`
- [ ] שליפת נתונים: הכנסות (חשבוניות) vs הוצאות (שעות עבודה × תעריף, הוצאות ישירות) per-client
- [ ] Prompt שמנתח ומחזיר:
```json
{
  "clients": [
    {
      "name": "יוסי כהן",
      "revenue": 15000,
      "estimatedCost": 12000,
      "profitMargin": 20,
      "status": "low_margin",
      "recommendation": "שקול להעלות תמחור ב-15% או לצמצם שעות"
    }
  ],
  "overallInsight": "3 מתוך 10 לקוחות לא רווחיים. המלצה: העלה מחירים ל-...",
  "topAction": "לקוח X מפסיד ₪2,000/חודש — דבר איתו על תמחור חדש"
}
```

### UI
- [ ] טאב "רווחיות" חדש ב-Finance Dashboard
- [ ] טבלה עם רווחיות per-client: הכנסה, עלות משוערת, מרג'ין
- [ ] Badge: רווחי (ירוק), גבולי (כתום), מפסיד (אדום)
- [ ] כרטיס AI Insight עם ההמלצה המרכזית

### UX
- [ ] ניתוח מתעדכן פעם בשבוע (או on-demand)
- [ ] התראה אוטומטית כשפרויקט הופך ללא רווחי
- [ ] כפתור "מה לעשות?" עם צעדים מומלצים

**מורכבות:** בינונית-גבוהה (צריך לחשב עלויות, לא רק הכנסות)
**זמן משוער:** 2-3 ימים
**הערה:** העלות משוערת יכולה להיות מבוססת על שעות עבודה שרשומות ב-Nexus × תעריף שעתי ממוצע

## ⚠️ Feature 3: חיזוי תזרים מזומנים (Cash Flow Prediction) — **RISKY**

**מודול:** Finance
**מה חסר:** אין חיזוי תזרים מזומנים לחודשים קדימה

### 🚨 אזהרה: פיצ'ר מסוכן
- **סיכון משפטי:** אם ה-AI טועה והלקוח בונה על כסף שלא נכנס — הוא יאשים אותך
- **דורש דאטה:** צריך 12+ חודשי היסטוריה כדי להיות מדויק
- **לא מתאים לעסקים חדשים:** אין מספיק נתונים
- **המלצה:** לבנות רק אם יש disclaimer ברור + רמת ביטחון נמוכה

**מה צריך לבנות:**

### Backend
- [ ] Server Action: `predictCashFlow` ב-`app/actions/finance-actions.ts`
- [ ] שליפת נתוני 6 חודשים אחרונים: הכנסות חודשיות, הוצאות, חובות פתוחים
- [ ] Prompt שמנתח מגמות ומחזיר:
```json
{
  "currentMonth": { "income": 45000, "expenses": 32000, "net": 13000 },
  "prediction3Months": { "income": 52000, "expenses": 35000, "net": 17000 },
  "trend": "positive",
  "confidence": "medium",
  "risks": ["3 חשבוניות פתוחות מעל 60 יום (₪8,500)", "לקוח X לא שילם חודשיים"],
  "recommendation": "שלח תזכורת ל-3 חשבוניות פתוחות. תזרים צפוי חיובי אם ייגבו."
}
```

### UI
- [ ] גרף תזרים מזומנים (3-6 חודשים קדימה) ב-Finance Dashboard
- [ ] כרטיס "חיזוי AI" עם מגמה (חיובי/שלילי/יציב)
- [ ] רשימת סיכונים עם Badge צבעוני
- [ ] כפתור "מה לעשות?" עם צעדים

### UX
- [ ] חישוב אוטומטי פעם בשבוע
- [ ] התראה כשחיזוי שלילי (תזרים צפוי שלילי)

**מורכבות:** בינונית (מבוסס על נתונים קיימים)
**זמן משוער:** 2-3 ימים

1. **כל פיצ'ר AI חייב להיות כנה** — אם אין מספיק נתונים, להגיד "אין מספיק נתונים לניתוח"
2. **אף פעם לא להמציא מספרים** — אם ה-AI לא בטוח, להגיד "רמת ביטחון: נמוכה"
3. **שקיפות** — להראות למשתמש מאיפה ה-AI שלף את המידע
4. **שימוש ב-AIService הקיים** — כל הפיצ'רים עוברים דרך AIService.ts
5. **אבטחה** — כל הנתונים מסוננים דרך ai-security.ts
6. **Credits** — כל שימוש גורע מה-token budget של הארגון
7. **🚨 אסור להשתמש ב-`as any`** — קוד חייב להיות type-safe. השתמש בטיפוסים מדויקים או unknown + type guards
