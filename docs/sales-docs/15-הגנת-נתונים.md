# 🛡️ הגנת נתונים ופרטיות

## Data Encryption

### בשידור (In Transit)
**TLS 1.3:**
- כל התקשורת מוצפנת
- Certificate: Let's Encrypt
- Grade A+ (SSL Labs)

```
Client → HTTPS → Server
לא ניתן לצותת!
```

### באחסון (At Rest)
**PostgreSQL Encryption:**
- Disk-level encryption
- AES-256
- Managed by provider (Vercel/Supabase)

**File Storage:**
- Supabase Storage
- Encrypted at rest
- Encrypted in transit

## Backup & Recovery

### גיבוי אוטומטי

**שמירה:**

## GDPR Compliance

### זכויות משתמש

**1. Right to Access**
```
משתמש יכול לבקש: "תן לי את כל הנתונים שלי"
זמן מענה: 30 יום
פורמט: JSON/CSV
```

**2. Right to Erasure (Right to be Forgotten)**
```
משתמש יכול לבקש: "מחק את כל הנתונים שלי"
זמן ביצוע: 7 ימים
מחיקה: סופית ובלתי הפיכה
```

**3. Right to Rectification**
```
משתמש יכול לבקש: "תקן נתון שגוי"
זמן ביצוע: מיידי
```

**4. Right to Portability**
```
משתמש יכול לבקש: "תן לי את הנתונים בפורמט שימושי"
פורמט: CSV/JSON
זמן: 24 שעות
```

### Data Processing Agreement (DPA)

**זמין ללקוחות:**
- חוזה DPA סטנדרטי
- תנאי GDPR
- Sub-processors list

## Privacy by Design

### מינימיזציה

**אוספים רק מה שצריך:**
- שם, מייל (חובה)
- טלפון (אופציונלי)
- לא אוספים: מיקום, גיל, מגדר (אלא אם נדרש לעסק)

### Pseudonymization

**מזהים:**
- UUID במקום שמות
- Hash במקום סיסמאות
- Masked data בלוגים

**דוגמה:**
```
Log: user_123 deleted lead_456
לא: John Doe deleted ABC Corp
```

## Data Breach Protocol

**אם קורה פריצה/דליפה:**

**1. זיהוי (Detection)**
- תוך דקות: התראה אוטומטית
- ניתוח היקף

**2. הכלה (Containment)**
- תוך שעה: סגירת גישות
- בידוד מערכות מושפעות

**3. חקירה (Investigation)**
- תוך 24 שעות: הבנת הנזק
- מה נחשף?
- כמה משתמשים הושפעו?

**4. דיווח (Notification)**
- תוך 72 שעות: דיווח לרשויות (GDPR)
- תוך 7 ימים: הודעה למשתמשים מושפעים

**5. תיקון (Remediation)**
- Patch אבטחה
- שינוי סיסמאות
- Lessons learned
