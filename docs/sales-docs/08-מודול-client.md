# 👥 Client — ניהול לקוחות ופורטלים

> **מחיר:** ₪149/חודש (סולו) | כלול בחבילת שיווק, חבילת הכל  
> **משתמשים:** 1 (סולו) או 5 (בחבילה)  
> 🎁 + מודול Finance במתנה לכל לקוח  
> **תפקיד:** Portal לקוחות, ניהול מסמכים, Workflows, סשנים ותקשורת דו-כיוונית. AI מנתח סנטימנט לקוח, מספק מענה אוטומטי, ומחזה שביעות רצון.

---

## מסכים ופיצ'רים

### 1. Portal לקוחות ⭐

| פיצ'ר | תיאור |
|--------|--------|
| **כניסה ייעודית** | הלקוח מקבל לינק הזמנה → נרשם → נכנס ל-Portal |
| **צפייה במסמכים** | רואה רק את המסמכים שלו |
| **העלאת קבצים** | הלקוח מעלה קבצים (חוזים, תמונות...) |
| **תקשורת** | הודעות דו-כיווניות (לקוח ↔ ספק) |
| **אישורים** | הלקוח מאשר/דוחה תוכן/הצעות |
| **Service Tiers** | רמות שירות שונות לכל לקוח |

**תהליך הזמנת לקוח:**
```
1. צור הזמנה (ClientPortalInvite)
2. לקוח מקבל מייל עם לינק
3. לקוח נרשם (ClientPortalUser)
4. לקוח נכנס ל-Portal
5. רואה רק מה ששייך אליו (tenant isolation)
```

### 2. ניהול קבצים ומסמכים

| פיצ'ר | תיאור |
|--------|--------|
| **העלאה** | Drag & Drop / בחירה |
| **תיקיות** | ארגון לפי קטגוריה |
| **שיתוף מאובטח** | רק למי שמורשה |
| **גרסאות** | `ClientDocumentFile` — מספר גרסאות לכל מסמך |
| **סוגי קבצים** | PDF, Word, Excel, תמונות, וידאו |

### 3. Workflows — תהליכי עבודה

| פיצ'ר | תיאור |
|--------|--------|
| **שלבים מוגדרים** | `MisradWorkflowStage` — סדר שלבים |
| **מעבר בין שלבים** | ידני או אוטומטי |
| **אישורים** | שלב דורש אישור לפני מעבר |
| **תבניות** | Blueprint — תבנית workflow שחוזרת על עצמה |
| **פריטים** | `MisradWorkflowItem` — משימות בתוך כל שלב |

**דוגמה: Workflow פרויקט עיצוב**
```
שלב 1: Brief (לקוח ממלא טופס)
  ↓
שלב 2: עיצוב ראשוני (המעצב מעלה)
  ↓
שלב 3: אישור לקוח (Portal — אישור/הערות)
  ↓
שלב 4: תיקונים
  ↓
שלב 5: אישור סופי
  ↓
שלב 6: מסירה (קבצים סופיים)
```

### 4. סשנים / פגישות

| פיצ'ר | תיאור |
|--------|--------|
| **תיעוד פגישה** | תאריך, משתתפים, סיכום |
| **הערות** | רישום חופשי |
| **משימות נגזרות** | צור משימות מתוך הסשן |
| **קבצים** | צירוף קבצים לסשן |

### 5. טפסים מותאמים

| פיצ'ר | תיאור |
|--------|--------|
| **Form Builder** | `MisradFormTemplate` — בנה טופס |
| **שלבים** | `MisradFormStep` — טופס מרובה שלבים |
| **שדות** | `MisradFormField` — text, number, select, file... |
| **תגובות** | `MisradFormResponse` — תשובות הלקוח |

### 6. AI לניהול לקוחות ⭐

| יכולת | תיאור | דוגמה |
|--------|--------|--------|
| **ניתוח סנטימנט** | AI מנתח את הטון של הודעות הלקוח | "לקוח X כועס — עדיפות גבוהה אוטומטית" |
| **מענה אוטומטי** | תשובות אוטומטיות לשאלות נפוצות | חוסך זמן תמיכה |
| **חיזוי שביעות רצון** | AI מזהה לקוחות שעומדים לעזוב | "שביעות רצון ירדה ל-50% — דבר איתו" |
| **המלצות FAQ** | AI מזהה שאלות חוזרות | "לקוח שאל את זה 3 פעמים — תוסיף ל-FAQ" |

## תהליכי עבודה

### תהליך: Onboarding לקוח חדש
```
1. צור לקוח חדש (ClientClient)
2. הגדר Service Tier (רמת שירות)
3. שלח הזמנה ל-Portal
4. לקוח נרשם ונכנס
5. שלח טופס Brief (Form)
6. צור Workflow
7. התחל עבודה
```

### תהליך: אישור תוכן (סוכנות שיווק)
```
1. מנהל תוכן מעלה פוסט (Social)
2. פוסט מופיע ב-Portal של הלקוח
3. לקוח מאשר / מבקש שינויים
4. מנהל תוכן מתקן
5. לקוח מאשר סופית
6. פוסט מתפרסם
```
