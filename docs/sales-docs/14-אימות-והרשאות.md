# 🔐 אימות והרשאות

## Clerk - פלטפורמת אימות

**מה זה:**
שירות אימות מנוהל לאפליקציות מודרניות.

**מה זה מספק:**
- ✅ Email/Password
- ✅ Social Login (Google, Apple, Microsoft)
- ✅ 2FA (Two-Factor Authentication)
- ✅ Password Reset
- ✅ Session Management
- ✅ Rate Limiting
- ✅ Brute Force Protection

---

## Authentication Flow

```
1. משתמש → Clerk Login Page
2. הזנת פרטים (Email + Password)
3. Clerk → אימות
4. Clerk → JWT Token
5. Token → Next.js Middleware
6. Middleware → Session
7. Session → getOrganizationId()
8. organizationId → Database Query
```

**זמן:** 2-3 שניות

## Session Management

**Token:**
- JWT (JSON Web Token)

**בטיחות:**

## RBAC - Role-Based Access Control

### תפקידים (Roles)

**1. Admin**
- כל ההרשאות
- ניהול משתמשים
- הגדרות מערכת
- מחיקת נתונים

**2. Manager**
- צפייה בכל הנתונים
- יצירה ועריכה
- לא מחיקה קריטית
- לא הגדרות מערכת

**3. Member**
- צפייה בנתונים שלו
- יצירה ועריכה בסיסית
- לא צפייה במשכורות
- לא מחיקה

**4. Viewer**
- צפייה בלבד
- לא עריכה
- לא יצירה
- לא מחיקה

## בדיקת הרשאות - קוד

```typescript
// app/actions/tasks.ts
'use server';

export async function deleteTask(taskId: string) {
  // 1. אימות
  const session = await auth();
  if (!session) throw new Error('Unauthorized');
  
  // 2. בדיקת הרשאה
  const hasPermission = await checkPermission(
    session.userId,
    'delete_task'
  );
  
  if (!hasPermission) {
    throw new Error('Forbidden: אין הרשאה למחוק משימות');
  }
  
  // 3. ביצוע
  await prisma.task.delete({ where: { id: taskId } });
}
```

## API Key Authentication

**לשימוש ב-API:**

```bash
curl -X POST https://misrad-ai.app/api/leads \
  -H "Authorization: Bearer sk_live_abc123..." \
  -d '{"name": "John Doe"}'
```

**יצירת API Key:**
1. הגדרות → API Keys
2. יצירת מפתח חדש
3. שמירה (מוצג פעם אחת!)
4. שימוש בכל קריאה

**אבטחה:**
- Prefix: `sk_live_` או `sk_test_`
- Hash: SHA-256
- Expiration: ללא (עד ביטול)
- Rate Limit: 1000/שעה

## 2FA (Two-Factor Authentication)

**מה זה:**
שכבת אבטחה נוספת.

**איך זה עובד:**
1. התחברות עם סיסמה
2. קוד נשלח ל-SMS/Email/App
3. הזנת קוד
4. גישה מאושרת

**מומלץ ל:**
- Admins (חובה!)
- כל מי שיש לו גישה לנתונים רגישים

---

## Session Timeout

**חוסר פעילות:**
- 30 דקות → התנתקות אוטומטית
- 24 שעות → Refresh Token פג תוקף

**אפשר להאריך:**
הגדרות → אבטחה → Session Timeout

## Password Policy

**דרישות מינימום:**
- 8 תווים
- אות גדולה
- אות קטנה
- מספר
- תו מיוחד (!, @, #...)

**מומלץ:**
- 12+ תווים
- Password Manager
- ייחודי לכל שירות
