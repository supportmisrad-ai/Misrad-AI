## אינטגרציות קיימות — פירוט

### 1. Google Calendar ✅

| מאפיין | ערך |
|--------|------|
| **סוג** | OAuth 2.0 דו-כיווני |
| **מודול** | Client |
| **מה עושה** | סנכרון אירועים, תזכורות, זמינות |

**איך להפעיל:**
```
1. הגדרות → אינטגרציות → Google Calendar
2. "Connect" → מסך אישור Google
3. אישור הרשאות (calendar.events, calendar.readonly)
4. סנכרון אוטומטי מתחיל
```

**טכנולוגיה:**
```typescript
// lib/integrations/google-calendar.ts
import { google } from 'googleapis';

async function syncCalendar(userId: string) {
  const auth = await getGoogleAuth(userId);
  const calendar = google.calendar({ version: 'v3', auth });
  
  const events = await calendar.events.list({
    calendarId: 'primary',
    timeMin: new Date().toISOString(),
    maxResults: 50,
  });
  
  // סנכרון דו-כיווני
  await syncToMisrad(events.data.items);
  await syncFromMisrad(userId);
}
```

**Log:** `scale_calendar_sync_log` — תיעוד כל סנכרון

### 5. Morning (חשבונית ירוקה) ✅

| מאפיין | ערך |
|--------|------|
| **סוג** | REST API |
| **מודול** | Finance |
| **מה עושה** | חשבוניות מס, קבלות, סנכרון דו-כיווני, מסמכי מע"מ |

> 💡 מודול Finance הוא **שכבת אינטגרציה** — הוא לא מערכת הנהלת חשבונות. הוא מתחבר לחברת החשבוניות שלך דרך API.

**טכנולוגיה:**
```typescript
// lib/integrations/green-invoice.ts
async function createGreenInvoice(invoiceData) {
  const response = await fetch('https://api.greeninvoice.co.il/api/v1/documents', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(invoiceData),
  });
  return response.json();
}
```

**דרישה:** חשבון Morning (חשבונית ירוקה) פעיל + API Key

### 5b. אינטגרציות Finance נוספות

המערכת תומכת ארכיטקטורלית באינטגרציות נוספות עם ספקי חשבוניות וסליקה:

**ספקי חשבוניות:** iCount, חשבוניות אונליין, סמיט, EZcount, רווחית, Grow  
**ספקי סליקה:** PayMe, משולם, CardCom, Tranzila, Hyp  
**מערכות ERP:** חשבשבת, פריורטי

כרגע רק **Morning (חשבונית ירוקה)** מחובר באופן מלא עם API ו-UI פעילים.

> � **הערה טכנית:** הוספת אינטגרציה חדשה דורשת API credentials מהספק, יצירת `lib/integrations/[provider].ts`, ועדכון UI ב-`IntegrationsView.tsx`.

### 6. Webhooks (Svix) ✅

| מאפיין | ערך |
|--------|------|
| **סוג** | Outbound (המערכת שולחת) |
| **מודול** | Core |
| **מה עושה** | שליחת אירועים למערכות חיצוניות |

**אירועים זמינים:**

| אירוע | מתי |
|--------|------|
| `lead.created` | ליד חדש נוצר |
| `lead.stage_changed` | ליד עבר שלב |
| `task.created` | משימה חדשה |
| `task.completed` | משימה הושלמה |
| `invoice.created` | חשבונית חדשה |
| `invoice.paid` | חשבונית שולמה |
| `work_order.completed` | קריאה הושלמה |

**Delivery Guarantees:**
- Retry אוטומטי (3 ניסיונות)
- Signature verification
- Delivery status tracking

**הגדרה:**
```
הגדרות → Webhooks → הוסף Endpoint
URL: https://your-app.com/webhook
Events: [lead.created, task.completed]
Secret: whsec_abc123...
```

## REST API

### Authentication

```bash
# API Key
curl -X POST https://misrad-ai.app/api/leads \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"full_name": "John Doe", "phone": "050-1234567"}'
```

### Endpoints עיקריים

| Method | Endpoint | תיאור |
|--------|----------|--------|
| `POST` | `/api/leads` | צור ליד חדש |
| `GET` | `/api/leads` | רשימת לידים |
| `PUT` | `/api/leads/:id` | עדכן ליד |
| `POST` | `/api/tasks` | צור משימה |
| `GET` | `/api/tasks` | רשימת משימות |
| `POST` | `/api/invoices` | צור חשבונית |
| `POST` | `/api/work-orders` | צור קריאת שירות |

### Rate Limits

| Tier | בקשות/שעה | בקשות/יום |
|------|-----------|-----------|
| Standard | 1,000 | 10,000 |
| Premium | 5,000 | 50,000 |

### Error Handling

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid API key",
    "status": 401
  }
}
```
