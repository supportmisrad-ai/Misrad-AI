Multi-Tenant — איך עובד הבידוד?

הרעיון

PostgreSQL אחד
├── Organization A → רואה רק את הנתונים שלה
├── Organization B → רואה רק את הנתונים שלה
└── Organization C → רואה רק את הנתונים שלה

כל ארגון = tenant נפרד. אי אפשר לראות נתונים של ארגון אחר.

4 שכבות בידוד

שכבה 1 — Organization ID בכל טבלה

-- כל טבלה עסקית מכילה עמודת organization_id
CREATE TABLE nexus_tasks (
  id              UUID PRIMARY KEY,
  organization_id UUID NOT NULL,   -- 👈 חובה!
  title           TEXT NOT NULL,
  ...
);
CREATE INDEX idx_tasks_org ON nexus_tasks(organization_id);

שכבה 2 — Tenant Guard (Middleware)

// lib/prisma-tenant-guard.ts
// כל query עובר דרך middleware שמוסיף WHERE organization_id = ?

export async function getTasks() {
  const orgId = await getOrganizationId(); // מה-Session
  
  return prisma.nexusTask.findMany({
    where: { organizationId: orgId } // 👈 אוטומטי, לא אפשר לעקוף
  });
}

שכבה 3 — Row Level Security (PostgreSQL)

-- אפילו SQL ישיר עובר דרך RLS
CREATE POLICY "tasks_tenant_isolation"
  ON nexus_tasks FOR ALL
  USING (organization_id = current_setting('app.organization_id')::uuid);

ALTER TABLE nexus_tasks ENABLE ROW LEVEL SECURITY;

שכבה 4 — API Guards + Audit

'use server';
export async function createTask(data: TaskInput) {
  const session = await auth();          // 1. מי אתה?
  const orgId = await getOrgId();        // 2. לאיזה ארגון?
  await checkPermission('create_task');  // 3. מותר לך?
  await checkRateLimit('create_task');   // 4. לא יותר מדי?
  
  const task = await prisma.nexusTask.create({
    data: { ...data, organizationId: orgId } // 5. orgId מאולץ
  });
  
  await logAudit('task.create', task.id);  // 6. תיעוד
  return task;
}

Server Actions — איך זה עובד?

מהם Server Actions?

פונקציות TypeScript שרצות אך ורק בשרת. הקוד לא נשלח לדפדפן.

// ✅ Server Action — רץ בשרת
// app/actions/leads.ts
'use server';

export async function createLead(name: string, phone: string) {
  // הלקוח לא רואה את הקוד הזה!
  const orgId = await getOrganizationId();
  return prisma.systemLead.create({
    data: { fullName: name, phone, organizationId: orgId }
  });
}

// ✅ Client Component — רץ בדפדפן
// components/LeadForm.tsx
'use client';
import { createLead } from '@/app/actions/leads';

export function LeadForm() {
  async function onSubmit(formData: FormData) {
    await createLead(formData.get('name'), formData.get('phone'));
  }
  return <form action={onSubmit}>...</form>;
}

שכבה 4: API Guards

מה זה: בדיקות בכל Server Action.

קוד:

export async function createTask(data) {
  // 1. אימות
  const session = await auth();
  if (!session) throw new Error('Unauthorized');
  
  // 2. ארגון
  const orgId = await getOrganizationId();
  if (!orgId) throw new Error('No org');
  
  // 3. הרשאות
  const canCreate = await checkPermission('create_task');
  if (!canCreate) throw new Error('Forbidden');
  
  // 4. Rate Limit
  const allowed = await checkRateLimit('create_task');
  if (!allowed) throw new Error('Rate limit');
  
  // 5. יצירה
  return prisma.task.create({
    data: { ...data, organizationId: orgId }
  });
}

'use server';

export async function deleteTask(taskId: string) {
  // 1. אימות
  const session = await auth();
  if (!session) throw new Error('Unauthorized');
  
  // 2. בדיקת הרשאה
  const hasPermission = await checkPermission(
    session.userId,
    'delete_task'
  );
  
  if (!hasPermission) {
    throw new Error('Forbidden: אין הרשאה למחוק משימות');
  }
  
  // 3. ביצוע
  await prisma.task.delete({ where: { id: taskId } });
}

// prepareAIContext — מתאים נתונים לפי הרשאות
function prepareAIContext(data, permissions) {
  const sanitized = sanitizeForAI(data);
  if (!permissions.includes('view_financials')) {
    delete sanitized.financialData;
  }
  return sanitized;
}

// validateAIResponse — בודק שה-AI לא הדליף
function validateAIResponse(response: string): boolean {
  const patterns = [/\b\d{9}\b/, /\b\d{4}[-\s]\d{4}\b/]; // מספרי ת.ז., כרטיסים
  return !patterns.some(p => p.test(response));
}

יזם סולו אחד בעזרת כלי AI. אני לא מסתיר את זה — זו נקודת מכירה, לא חולשה.

₪149/חודש למודול בודד, ₪499/חודש לכל 6 המודולים + 5 משתמשים.

כן, בכל עת, ללא התחייבות, ללא דמי ביטול.

Chrome, Firefox, Safari, Edge — גרסאות אחרונות.

PWA (Progressive Web App) — נפתח בדפדפן מובייל. אין אפליקציה ב-App Store.

Vercel מספק 99%+ uptime. אני מקבל התראה ומתקן. אין SLA רשמי.

איפה הנתונים שלי?

PostgreSQL — אירופה (Frankfurt) או ארה"ב (Oregon). אתה בוחר.

כן. TLS 1.3 בשידור, AES-256 באחסון.

האם יש API?

כן. REST API עם API Key authentication. 1000 requests/שעה.

כן. דרך Svix — delivery guarantees + retry logic.

7 שכבות: Clerk Auth, Tenant Guard, RLS, API Guards, AI Security, Audit Log, Encrypted Backup.

לא. אם צריך SOC 2 — לא מתאים.

רק המשתמשים בארגון שלך. Tenant Isolation מלא. אפילו אני (האדמין) צריך הרשאה מפורשת.
