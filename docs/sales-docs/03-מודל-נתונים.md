# 🗄️ מודל נתונים - Database Schema

## סקירה כללית

**PostgreSQL 16** עם **250+ טבלאות**

**חלוקה:**
- Nexus: ~40 טבלאות
- System: ~30 טבלאות
- Social: ~50 טבלאות
- Finance: ~15 טבלאות
- Client: ~25 טבלאות
- Operations: ~35 טבלאות
- Shared: ~55 טבלאות (auth, billing, audit)

---

## טבלאות מרכזיות

### Organizations (ארגונים)

```sql
social_organizations:
├── id (UUID)
├── name, slug, logo
├── owner_id → social_users
├── Module Flags:
│   ├── has_nexus (boolean)
│   ├── has_system
│   ├── has_social
│   ├── has_finance
│   ├── has_client
│   └── has_operations
├── Subscription:
│   ├── subscription_status (trial/active/past_due)
│   ├── subscription_plan
│   ├── trial_days (7)
│   └── seats_allowed
└── Settings:
    ├── ai_credits_balance_cents
    └── is_shabbat_protected
```

### Users (משתמשים)

```sql
nexus_users:
├── id, organization_id
├── name, email, phone, avatar
├── role, department
├── Status: online, last_seen_at
├── Performance: capacity, streak_days, weekly_score
└── Payment: hourly_rate, monthly_salary (🔒 רגיש!)
```

### Tasks (משימות)

```sql
nexus_tasks:
├── id, organization_id
├── title, description
├── status, priority
├── assignee_ids[] (מערך משתמשים)
├── due_date, time_spent
├── is_timer_running
└── messages[] (JSONB)
```

### Leads (לידים)

```sql
system_leads:
├── id, organization_id
├── full_name, company, email, phone
├── source, stage
├── value (decimal)
├── assigned_to
├── score (AI 0-100)
└── custom_fields (JSONB)
```

### Posts (פוסטים)

```sql
social_posts:
├── id, organization_id, client_id
├── content, media_url
├── status (draft/scheduled/published)
├── scheduled_at
└── Relationships:
    ├── platforms[] (Facebook, Instagram...)
    └── variations[] (וריאציות תוכן)
```

### Invoices (חשבוניות)

```sql
finance_invoices:
├── id, organization_id
├── client_name, client_email
├── invoice_number (unique)
├── amount, currency (ILS)
├── issue_date, due_date, paid_at
├── status (draft/sent/paid/overdue)
└── green_invoice_id (אינטגרציה)
```

### Work Orders (קריאות שירות)

```sql
operations_work_orders:
├── id, organization_id
├── title, description
├── client_name, client_phone
├── address, latitude, longitude
├── assigned_to (טכנאי)
├── status, priority
├── scheduled_at, completed_at
└── completion_signature_url (חתימת לקוח)
```

## Indexes (אינדקסים)

**כל טבלה:**
```sql
CREATE INDEX idx_xxx_org ON table_name(organization_id);
```

**ביצועים:**
```sql
CREATE INDEX idx_tasks_status ON nexus_tasks(status);
CREATE INDEX idx_tasks_due ON nexus_tasks(due_date);
CREATE INDEX idx_leads_stage ON system_leads(stage);
```

**Composite Indexes:**
```sql
CREATE INDEX idx_tasks_org_status 
ON nexus_tasks(organization_id, status);
```

## Relationships (קשרים)

### One-to-Many

```
Organization → Users (1:N)
Organization → Tasks (1:N)
User → Tasks (1:N) [creator]
Task → Users (N:M) [assignees]
```

### Many-to-Many

```
Task ←→ Users (assignee_ids[])
Post ←→ Platforms (social_post_platforms)
```

## JSONB Fields

**משתמשים בהם ל:**
- Custom Fields (שדות מותאמים)
- Settings (הגדרות)
- Messages (תקשורת)
- Metadata (מטא-דאטה)

**דוגמה:**
```sql
-- custom_fields
{
  "industry": "tech",
  "employees": "50-100",
  "budget": "high"
}

-- messages
[
  {"from": "user_123", "text": "צריך לסיים", "at": "2026-02-10T10:00:00Z"},
  {"from": "user_456", "text": "בסדר", "at": "2026-02-10T10:05:00Z"}
]
```

## Soft Delete

**רוב הטבלאות:**
```sql
deleted_at TIMESTAMPTZ NULL
```

**מחיקה:**
```sql
UPDATE tasks SET deleted_at = NOW() WHERE id = '...';
```

**שאילתה:**
```sql
SELECT * FROM tasks WHERE deleted_at IS NULL;
```
