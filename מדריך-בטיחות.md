# 🛡️ מדריך בטיחות דאטאבייס

## ⚠️ פקודות שחסומות (לא ניתן להריץ)

הפקודות הבאות **חסומות** כדי למנוע מחיקת נתונים בטעות:

```bash
# ❌ חסום!
npm run prisma:migrate:dev
npm run prisma:migrate:reset
npm run db:run:sql
```

---

## ✅ פקודות בטוחות (במקום)

### 1. לבדוק את מצב הדאטאבייס
```bash
npm run migrate:check
npm run db:check:connection
npm run db:check:migrations
```

### 2. ליצור מיגריישן חדש (בטוח)
```bash
npm run migrate:safe <שם-מיגריישן>
```

**מה זה עושה:**
- בודק אם יש נתונים בדאטאבייס
- **שואל אותך** אם יצרת גיבוי
- יוצר את קובץ המיגריישן
- מראה לך לבדוק את ה-SQL
- **שואל שוב** אם להריץ
- רק אז מריץ את המיגריישן

### 3. להריץ קובץ SQL (בטוח)
```bash
npm run sql:safe scripts/my-script.sql
```

**מה זה עושה:**
- בודק אם יש פקודות מסוכנות (DROP, TRUNCATE, DELETE)
- **מזהיר אותך** אם יש
- דורש **3 אישורים** לפני הרצה:
  1. "האם אתה בטוח לחלוטין?"
  2. "האם יצרת גיבוי?"
  3. "הקלד 'אני מאשר'"

### 4. ליצור משתמש וארגונים
```bash
npm run db:create:my-user    # יוצר את המשתמש שלך
npm run db:create:demos      # יוצר 13 ארגוני דמו
```

## � מערכת גיבויים אוטומטית

### יצירת גיבוי (פשוט!)
```bash
npm run db:backup
```

**מה זה עושה:**
- יוצר קובץ JSON עם כל הנתונים
- שומר ב-`backups/backup-[תאריך].json`
- שומר את 10 הגיבויים האחרונים
- מציג סיכום מלא

### שחזור מגיבוי
```bash
npm run db:restore
```

**מה זה עושה:**
- מראה לך רשימת כל הגיבויים
- מציג תאריך וכמה נתונים בכל אחד
- מבקש ממך לבחור איזה לשחזר
- **שואל אישור לפני שחזור**

### איפה הגיבויים?
```
c:\Projects\Misrad-AI\backups\
├── backup-2026-02-04T09-30-00.json
├── backup-2026-02-04T10-15-00.json
└── backup-2026-02-04T11-00-00.json
```

**הגיבויים לא עולים ל-Git** - הם רק אצלך במחשב.

## � תהליך בטוח לשינויי סכמה

### צעד 1: גיבוי
```bash
npm run db:backup
```

### צעד 2: ערוך את schema.prisma
פתח את `prisma/schema.prisma` ועשה את השינויים.

### צעד 3: בדוק מצב
```bash
npm run migrate:check
```

### צעד 4: צור מיגריישן בטוח
```bash
npm run migrate:safe add_new_field
```

הסקריפט ישאל:
- ✅ "האם יצרת גיבוי? (כן/לא)"
- ✅ "להריץ את המיגריישן עכשיו? (כן/לא)"

## 🚨 מה לעשות אם משהו השתבש

### אם מחקת נתונים בטעות:

**אופציה 1: שחזור מגיבוי מקומי (מהיר)**
```bash
npm run db:restore
```

**אופציה 2: Point-in-Time Recovery ב-Supabase (למקרים קשים)**
1. לך ל-Supabase Dashboard
2. Settings → Database → Backups
3. בחר נקודת זמן לפני המחיקה
4. לחץ Restore

**⚠️ חשוב:** Supabase שומר גיבויים יומיים אוטומטיים + Point-in-Time Recovery!

### אם יש migration תקוע:
1. לך ל-Supabase Dashboard
2. SQL Editor
3. הרץ:
```sql
SELECT * FROM _prisma_migrations WHERE finished_at IS NULL;
DELETE FROM _prisma_migrations WHERE finished_at IS NULL;
```

## 🗂️ מערכת הגיבויים - 3 שכבות הגנה

### שכבה 1: גיבויים מקומיים (אצלך במחשב)
```bash
npm run db:backup      # יוצר גיבוי JSON
npm run db:restore     # משחזר מגיבוי
```
- **מהיר** - שניות בודדות
- **קל לשימוש** - עם תפריט בעברית
- **10 גיבויים אחרונים** נשמרים

### שכבה 2: Supabase Daily Backups
- גיבוי אוטומטי כל יום
- נשמר 7 ימים (בתוכנית Free)
- נשמר 30+ ימים (בתוכניות Pro)
- גישה דרך: Settings → Database → Backups

### שכבה 3: Point-in-Time Recovery (PITR)
- Supabase שומר כל שינוי
- אפשר לחזור לכל נקודת זמן
- זמין בתוכניות Pro ומעלה
- מושלם למקרי חירום
