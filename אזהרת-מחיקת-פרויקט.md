# ⚠️ אזהרה קריטית - מחיקת פרויקט Supabase

## 🚨 סכנה מספר 1: מחיקת הפרויקט

אם מוחקים את הפרויקט ב-Supabase Dashboard:
- ❌ **כל הנתונים נמחקים לצמיתות**
- ❌ **אין ביטול** - זה סופי
- ❌ **גיבויי Supabase גם נמחקים**
- ❌ **לא ניתן לשחזר**

---

## 🛡️ הגנות שצריך להפעיל

### 1. הגנה ברמת Supabase

**הגדר סיסמה חזקה:**
- לך ל-Supabase Dashboard
- Settings → Account
- שנה סיסמה למשהו חזק + Two-Factor Authentication (2FA)

**הגבל גישה:**
- לא תיתן לאף אחד אחר גישה לפרויקט
- אם יש שותפים - הגדר להם הרשאות מוגבלות
- Settings → Team → רק Read-Only למי שלא חייב

### 2. הגנת הפרויקט עצמו

**ב-Supabase Dashboard:**
1. Project Settings → General
2. גלול ל-"Pause project"
3. **אל תלחץ "Delete project"** - זה בלתי הפיך!

**אין כפתור "נעל פרויקט"** ב-Supabase - צריך להיזהר ידנית.

---

## 💾 הגנה האמיתית: גיבויים חיצוניים

### אם הפרויקט נמחק - רק זה יעזור:

**1. גיבויים מקומיים (שיצרנו):**
```bash
npm run db:backup  # לפני כל שינוי חשוב
```
- ✅ שמור ב-`backups/` אצלך במחשב
- ✅ לא תלוי ב-Supabase
- ✅ אפשר לשחזר לפרויקט חדש

**2. גיבוי ל-Git (אופציונלי):**
```bash
# גיבוי נוסף - לדחוף ל-GitHub (private repo)
git add backups/
git commit -m "backup $(date)"
git push
```
⚠️ **רק אם ה-repo פרטי!** לא לשים נתונים רגישים ב-public.

**3. גיבוי לענן אחר:**
- Google Drive
- Dropbox
- OneDrive

## 🚨 מה לעשות אם הפרויקט נמחק

### אם זה קרה:

1. **אל תיבהל** - הגיבויים המקומיים שלך קיימים
2. **צור פרויקט Supabase חדש**
3. **עדכן את ה-DATABASE_URL ב-.env.local**
4. **הרץ:**
   ```bash
   npm run db:restore
   ```
5. **בחר את הגיבוי האחרון**
